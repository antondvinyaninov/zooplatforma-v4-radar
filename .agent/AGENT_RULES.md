# Правила поведения ИИ-Агента (ZooPlatform 4.0)

Этот файл содержит критически важные инструкции по взаимодействию с пользователем и кодовой базой проекта. Все агенты ОБЯЗАНЫ следовать этим правилам.

## 1. Взаимодействие с пользователем 🗣️

- **Никакой самодеятельности**: Если ИИ-агент хочет проявить инициативу (добавить новый функционал, изменить дизайн, внедрить UX-хелперы вроде «прицела» и т.д.), он **ОБЯЗАН** сначала спросить пользователя и получить его явное одобрение ("да", "давай", "ок").
- **Запрет на нежданные функции**: Запрещено внедрять элементы интерфейса или логику, которые не были прямо запрошены пользователем, даже если они кажутся «полезными».
- **Краткость и точность**: Ответы должны быть сфокусированы на решении текущей задачи без лишней воды.

## 2. Код и Архитектура 🏗️

- **VKUI Style**: Всегда использовать токены и переменные из VKUI (`var(--vkui--...)`) для обеспечения корректной работы тем.
- **Минимализм**: Придерживаться чистоты кода и удалять/комментировать неиспользуемые фрагменты (с объяснением причин).

## 3. Работа с памятью MemPalace 🧠

- **Глобальная память по запросу**: Если пользователь просит "вспомнить", "проверить в памяти", "посмотреть прошлые решения", "найти контекст" или явно ссылается на MemPalace, агент должен сначала проверить **глобальный MemPalace**, а не ограничиваться только текущим проектом.
- **Сначала широкий поиск, потом сужение**: По умолчанию сначала искать по всем доступным wing, а затем при необходимости сужать поиск до конкретного wing или room.
- **Проектный wing — только как уточнение**: `vk_mini_app_full` использовать как прицельный фильтр, когда запрос точно относится только к этому репозиторию или когда глобальный поиск уже дал нужный контекст.
