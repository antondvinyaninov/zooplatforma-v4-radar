# Работа кеша приложений

Чтобы предоставить пользователям комфортное взаимодействие с мини-приложениями для Android и iOS, платформа ВКонтакте использует кеш. Он помогает восстановить последний используемый экран мини-приложения, когда пользователь возвращается в это мини-приложение после выхода из него.

Кеш используется неявно, разработчику не надо заботиться о взаимодействии с ним. Библиотека [VK Bridge](/ru/bridge/overview) не содержит событий, связанных с кешем. Мы решили рассказать об использовании кеша, чтобы предоставить более полную картину работы и помочь понять, почему мини-приложение показывает тот или иной экран, а VK Bridge отправляет то или иное событие.

## [](<#Как это выглядит для пользователя>)Как это выглядит для пользователя

*   •
    
    Представьте, что пользователь получает ссылку на мини-приложение от друга. Эта ссылка может представлять собой адрес самого мини-приложения или адрес мини-приложения и экрана в нём. Например:
    
    *   •
        
        `https://vk.com/app12345` — ссылка, содержащая только адрес мини-приложения.
        
    *   •
        
        `https://vk.com/app12345/#/page1` — ссылка с указанием на экран в мини-приложении.
        
    
*   •
    
    Когда пользователь переходит по экранам мини-приложения, текущий URL мини-приложения может меняться.
    
*   •
    
    Если пользователь выйдет из мини-приложения, а потом нажмёт на изначальную ссылку, чтобы в него вернуться, то ВКонтакте запустит мини-приложение, но покажет последний экран, на котором был пользователь, а не тот экран, на который указывает ссылка.
    

Такой подход в большей мере соответствует ожиданиям пользователей и обеспечивает комфортную работу с мини-приложением.

Для реализации подобного подхода ВКонтакте использует кеш приложений.

## [](<#Как работает кеш>)Как работает кеш

1.  1.
    
    При запуске мини-приложения мобильное приложение ВКонтакте сохраняет информацию о URL, используемом для запуска.
    
2.  2.
    
    Когда пользователь выходит из мини-приложения любым способом — сворачивает мобильное приложение ВКонтакте, переключается в другое мобильное приложение, возвращается на главный экран, выходит из мини-приложения, нажав на кнопку «Закрыть» — платформа ВКонтакте выполняет следующие действия:
    
    *   •
        
        Отправляет мини-приложению событие [`VKWebAppViewHide`](/ru/bridge/VKWebAppViewHide).
        
        Вы можете использовать его, чтобы [приостановить действия](/ru/games/how-to/handle-minimize-and-restore-events), которые не должны выполняться в фоновом режиме, например воспроизведение музыки.
        
    *   •
        
        Помещает данные мини-приложения, включая текущий URL, в кеш.
        
    
3.  3.
    
    Информация в кеше хранится 15 минут.
    
    Если в течение этого периода пользователь вернётся в мини-приложение или запустит его снова, ВКонтакте будет использовать кеш.
    
    *   •
        
        Если для входа в мини-приложение используется **тот же самый URL**, который был использован для запуска ранее, ВКонтакте загрузит информацию из кеша. Визуально для пользователя это будет выглядеть так, как будто мини-приложение открылось на экране, который был активен в момент выхода.
        
        После запуска ВКонтакте сначала отправит мини-приложению событие [`VKWebAppChangeFragment`](/ru/bridge/VKWebAppChangeFragment), а затем [`VKWebAppViewRestore`](/ru/bridge/VKWebAppViewRestore). `VKWebAppChangeFragment` сообщает приложению об экране, на который надо перейти. Если во время отсутствия пользователя на экране произошли изменения, то это даст приложению возможность обновить экран. `VKWebAppViewRestore` сигнализирует о возврате пользователя в приложение.
        
    *   •
        
        Если для входа используется **другой URL**, то мини-приложение запустится заново.
        
    

## [](<#Какие приложения кешируются>)Какие приложения кешируются

Платформа ВКонтакте использует кеш и для мини-приложений, и для игр. Кеш имеет два слота, то есть хранит информацию для двух последних открытых мини-приложений или игр. Наши исследования показали, что этого достаточно для комфортной работы.
