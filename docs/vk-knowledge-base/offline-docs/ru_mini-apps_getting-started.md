# Первые шаги

> В связи с ростом количества пользователей, сообществ и сервисов ВКонтакте мы рекомендуем хранить идентификаторы пользователей и других необходимых значений в диапазоне `Int64` вместо `Int32`. Подробнее — в статье [Переход на LongID](/ru/general/long-id).

В этом разделе вы узнаете, как добавить ваше первое мини-приложение на площадки ВКонтакте и Одноклассники. Эта инструкция включает шаги создания приложения из шаблона, регистрацию и размещение на хостинге.

Подробнее о полном процессе создания приложения от идеи до монетизации — в нашем видеокурсе:

    ► [Разработка мини-приложений ВКонтакте](/ru/mini-apps/learning/course)

## [](<#О мини-приложениях>)О мини-приложениях

Мини-приложения ВКонтакте и в Одноклассниках — это веб-приложения, которые запускаются внутри интерфейса социальных сетей. Клиентская часть представляет собой обычные веб-страницы, а серверная может быть реализована на любой современной технологии разработки.

При регистрации мини-приложению автоматически присваивается идентификатор. Оно открывается по специальному адресу и загружается внутри ВКонтакте или Одноклассников.

Для взаимодействия с площадками мини-приложения используют их API — например, для запроса разрешений или доступа к данным пользователя. Чтобы упростить обмен данными, используйте библиотеку [VK Bridge](/ru/bridge/overview) — она работает на стороне клиента и позволяет отправлять запросы и получать ответы.

## [](<#Что потребуется>)Что потребуется

*   •
    
    Навыки работы с командной строкой. Используйте окно командной строки операционной системы либо редактор кода, в котором есть командная строка, например [Visual Studio Code](https://vk.com/away.php?to=https%3A%2F%2Fcode.visualstudio.com%2F).
    
*   •
    
    Node.js. Скачать и установить эту среду можно с [официального сайта nodejs.org](https://vk.com/away.php?to=https%3A%2F%2Fnodejs.org).
    
    > Мы рекомендуем использовать версию Node.js 18.0.0 или более позднюю.
    
*   •
    
    Мобильное устройство для подтверждения действий при регистрации мини-приложения и работе с его настройками.
    

## [](<#1. Создайте проект мини-приложения>)1\. Создайте проект мини-приложения

Начните с создания мини-приложения на своём компьютере. Вы можете применять любые среды разработки, языки и технологии, которые используются для создания HTML5-приложений.

Рассмотрим, как создать проект мини-приложения из шаблона с помощью пакета [create-vk-mini-app](/ru/mini-apps/getting-started/create-vk-mini-app). Результирующий проект представляет собой npm-пакет, в котором подключены все необходимые библиотеки. Функциональность этого мини-приложения включает два экрана с простой навигацией и кнопкой, которая показывает стикер.

1.  1.
    
    Откройте окно командной строки и перейдите в папку, в которой хотите создать проект мини-приложения:
    
    Командная строка
    
    `cd <путь-к-папке>`
    
2.  2.
    
    Запустите скрипт создания приложения из шаблона.
    
    Командная строка
    
    `npx @vkontakte/create-vk-mini-app`
    
    При первом использовании появится запрос на скачивание пакета @vkontakte/create-vk-mini-app. Нажмите **Enter**, чтобы подтвердить.
    
3.  3.
    
    Далее скрипт последовательно запросит следующие данные:
    
    *   •
        
        Название папки, в которой будут размещены файлы приложения. Для простоты введите точку — тогда проект будет создан в текущей папке. Вы также можете использовать предлагаемое имя — `mini-app`. В этом случае скрипт создаст для проекта подпапку `mini-app` в текущей папке.
        
        По умолчанию имя папки будет использовано и для именования результирующего пакета. Если же имя не может быть использовано для именования пакетов, скрипт попросит вас ввести название проекта. Это название будет указано в файле `package.json` из корневой папки проекта после создания.
        
    *   •
        
        Язык программирования: TypeScript или JavaScript. Выберите тот язык, который вам знаком. Это не принципиально для создаваемого проекта.
        
    *   •
        
        Структурный шаблон — набор библиотек, которые будут подключены к приложению. Выберите рекомендуемый вариант — **VKUI+Bridge+Router**.
        
    *   •
        
        Скрипт создаст файлы приложения в указанной вами папке и покажет дальнейшие инструкции. Выполним только первые две из них.
        
        *   •
            
            Если вы указали имя проекта при его создании, перейдите в созданную папку проекта:
            
            Командная строка
            
            `cd <папка-проекта>`
            
        *   •
            
            Установите все пакеты, указанные в файле `package.json` вашего проекта:
            
            Командная строка
            
            `npm install`
            
            Подождите, пока пакеты установятся. Выполнять `npm run start` пока не нужно.

Обратите внимание, что при вызове create-vk-mini-app вы можете добавить дополнительные параметры командной строки, чтобы указать желаемые характеристики приложения. Подробнее — в [описании пакета create-vk-mini-app](/ru/mini-apps/getting-started/create-vk-mini-app).

**Важно!** Для корректной работы мини-приложения к нему необходимо подключить библиотеку [VK Bridge](/ru/bridge/getting-started), а также вызвать событие инициализации [`VKWebAppInit`](/ru/bridge/VKWebAppInit) в начале работы мини-приложения. Если не вызвать событие, мини-приложение не сможет взаимодействовать с площадками ВКонтакте и Одноклассники.

Если вы будете создавать мини-приложение без использования пакета create-vk-mini-app, подключите VK Bridge и самостоятельно в коде приложения вызовите событие инициализации. Подробнее — в [описании библиотеки VK Bridge](/ru/bridge/getting-started).

## [](<#2. Зарегистрируйте мини-приложение>)2\. Зарегистрируйте мини-приложение

1.  1.
    
    Откройте [десктопную версию сайта ВКонтакте](https://vk.com/away.php?to=https%3A%2F%2Fvk.com) и войдите в свой аккаунт.
    
2.  2.
    
    Откройте [страницу со списком приложений](https://vk.com/away.php?to=https%3A%2F%2Fdev.vk.com%2Fru%2Fadmin%2Fapps-list) либо перейдите на портал разработчиков [dev.vk.com](https://vk.com/away.php?to=https%3A%2F%2Fdev.vk.com) и выберите **Приложения** в верхнем меню.
    
3.  3.
    
    В левом меню нажмите **Создать приложение**.
    
    ![Создание мини-приложения](https://sun9-47.userapi.com/1KfRkjurMf3je6d6DULsGj0ATWqYtzApooxjpg/fyVkxAtTTOA.jpg "Создание мини-приложения")Создание мини-приложения
    
4.  4.
    
    Заполните свойства мини-приложения:
    
    *   •
        
        В поле **Тип** выберите **Мини-приложение**.
        
    *   •
        
        Укажите название приложения. Оно может содержать только буквы, цифры, дефисы, пробелы, двоеточия и тире. Вы можете указать любое имя и изменить его позже в разделе [Настройки → Информация](/ru/mini-apps/settings/general/information).
        
    *   •
        
        Выберите категорию мини-приложения.
        
    *   •
        
        Нажмите **Создать**.
        
        Во всплывающем окне **Подтверждение действия** выберите способ подтверждения и одобрите действие.
        
        > Если вы выбрали подтверждение с помощью мобильного приложения ВКонтакте, push-уведомление придёт, только если приложение запущено на вашем смартфоне.
        
    

Регистрация приложения завершена. Откроется панель управления приложением. Идентификатор мини-приложения генерируется автоматически при регистрации. Он необходим для запуска и других операций с приложением.

![Панель управления приложением](https://sun9-43.userapi.com/BOXLc26JIONrXVf9JD90NBfgCy3V66o75JEu3A/os7Se_W4-Y4.jpg "Панель управления приложением")Панель управления приложением

Чтобы разместить мини-приложение в Одноклассниках, перейдите в раздел [Настройки → Другие площадки](/ru/mini-apps/settings/general/other-sites) и нажмите **Привязать OK ID**. Подробнее о подключении мини-приложения к Одноклассникам — в разделе [Публикация в Одноклассниках](/ru/ok/publishing#%D0%A8%D0%B0%D0%B3%202.%20%D0%9F%D0%BE%D0%B4%D0%BA%D0%BB%D1%8E%D1%87%D0%B8%D1%82%D0%B5%20%D0%BF%D1%80%D0%B8%D0%BB%D0%BE%D0%B6%D0%B5%D0%BD%D0%B8%D0%B5%20%D0%BA%20%D0%9E%D0%B4%D0%BD%D0%BE%D0%BA%D0%BB%D0%B0%D1%81%D1%81%D0%BD%D0%B8%D0%BA%D0%B0%D0%BC).

## [](<#3. Запустите мини-приложение через туннель>)3\. Запустите мини-приложение через туннель

Ваше мини-приложение пока находится на вашем компьютере. Чтобы сделать его доступным извне, необходимо настроить специальное соединение — туннель. Для создания такого соединения мы разработали библиотеку — [VK Tunnel](/ru/libraries/tunnel).

**Из-за технических работ с 2 октября 2025 года VK Tunnel недоступен**

При подключении сервис вернёт ошибку. Для отладки и демонстрации ваших веб-приложений рекомендуем временно использовать сторонние решения.

Вы можете пропустить этот шаг и сразу разместить мини-приложение на хостинге.

#### [](<#3.1. Подготовьте проект>)3.1. Подготовьте проект

1.  1.
    
    Если вы работаете в Windows, установите дополнительные инструменты разработчика.
    
    Запустите **Пуск → Все приложения → Node.js → Install Additional Tools for Node.js** и следуйте инструкциям на экране.
    
    Альтернативный способ:
    
    *   •
        
        Загрузите установщик Visual Studio Build Tools: [https://aka.ms/vs/17/release/vs\_BuildTools.exe](https://vk.com/away.php?to=https%3A%2F%2Faka.ms%2Fvs%2F17%2Frelease%2Fvs_BuildTools.exe).
        
    *   •
        
        Запустите его и выберите для установки компонентов **Средства сборки Node.js** и **Разработка классических приложений на C++**.
        
    
2.  2.
    
    Установите VK Tunnel. Для этого перейдите в папку проекта мини-приложения и выполните следующую команду:
    
    Командная строка
    
    `npm install @vkontakte/vk-tunnel --include=dev`
    
3.  3.
    
    Добавьте в файл `package.json` вашего проекта скрипт `tunnel` и сохраните изменения:
    
    JSON
    
    `"scripts": { ... "tunnel": "vk-tunnel --insecure=1 --http-protocol=http --ws-protocol=ws --host=0.0.0.0 --port=5173" }`
    
    В приложении, созданном из шаблона с помощью create-vk-mini-app, скрипт `tunnel` уже добавлен. Если параметры скрипта отличаются от указанных, обновите их.
    

#### [](<#3.2. Запустите мини-приложение локально>)3.2. Запустите мини-приложение локально

Локальный запуск необходим для дальнейшего запуска внутри ВКонтакте.

1.  1.
    
    В окне командной строки перейдите в папку проекта мини-приложения:
    
    Командная строка
    
    `cd <путь-к-папке-проекта>`
    
2.  2.
    
    Выполните следующую команду:
    
    Командная строка
    
    `npm run start -- --host`
    
    Она запустит сервер мини-приложения на вашем компьютере.
    
    > **Важно!** Не пропустите параметр `-- --host`. Он необходим для работы.
    
    Если всё в порядке, вы увидите следующее в окне командной строки:
    
    Окно командной строки
    
    `VITE v5.2.10 ready in 441 ms ➜ Local: http://localhost:5173/ ➜ Network: http://100.75.14.91:5173/ ➜ press h + enter to show help`
    

Не закрывайте окно командной строки, иначе сервер мини-приложения остановится. Он должен работать для последующего запуска.

#### [](<#3.4. Обновите настройки мини-приложения>)3.4. Обновите настройки мини-приложения

1.  1.
    
    В браузере перейдите в [настройки мини-приложения](/ru/mini-apps/settings/overview).
    
2.  2.
    
    Откройте раздел [Настройки → Размещение](/ru/mini-apps/settings/general/placement). В блоке **Десктопная версии сайта** включите **Режим разработки** и укажите полученную на предыдущем шаге ссылку как URL для разработки. То же самое можно сделать и для других платформ.
    

![Настройка «Режим разработки»](https://sun9-29.userapi.com/yaUuvRLFqHa1bxbBGFAl-Hr_cJAoTFhQKnDVfw/bttx8SRTK3w.jpg "Настройка «Режим разработки»")Настройка «Режим разработки»

URL, которые вы указываете в разделе [Размещение](/ru/mini-apps/settings/general/placement), используются для запуска мини-приложения. Именно этот адрес будет передан в элемент `<iframe>` или компонент WebView. Когда включён режим разработки, для администраторов приложения будет использован адрес, указанный для этого режима. Для остальных пользователей приложение будет загружаться по адресу, указанному в поле **URL**.

#### [](#Замечания)Замечания

*   •
    
    Мини-приложение будет работать, только пока оно и VK Tunnel запущены на вашем локальном компьютере. Чтобы мини-приложение работало автономно, вам потребуется разместить его на хостинге.
    
*   •

## [](<#4. Разместите мини-приложение на хостинге>)4\. Разместите мини-приложение на хостинге

Чтобы мини-приложение работало автономно и не зависело от работы туннеля, приложение необходимо разместить на постоянно доступном сервере. Возможные пути решения:

*   •
    
    Воспользоваться услугами хостинговых компаний. Вам нужно выбрать тариф с поддержкой HTTPS.
    
*   •
    
    Разместить мини-приложение в [облаке VK Cloud](https://vk.com/away.php?to=https%3A%2F%2Fcloud.vk.com%2Fdocs%2Fru%2Fadditionals%2Fstart).
    
*   •
    
    Использовать хостинг, который мы предлагаем для мини-приложений. Файлы будут размещаться в CDN-инфраструктуре компании. Решение бесплатно, но есть ограничения на размер выкладываемых файлов и частоту обновлений. Давайте рассмотрим, как им можно воспользоваться.
    
    Обращаем ваше внимание, что мы предоставляем место для хранения файлов, но не вычислительные мощности для работы сервера. Если вашему приложению нужно выполнять обработку данных на сервере, воспользуйтесь другими возможностями для хостинга.
    

Для работы с хостингом мы создали библиотеку [vk-miniapps-deploy](https://vk.com/away.php?to=https%3A%2F%2Fgithub.com%2FVKCOM%2Fvk-miniapps-deploy). Она включена в ваше мини-приложение, потому что оно было создано с помощью create-vk-mini-app.

Если по какой-то причине библиотеки нет в вашем проекте, вы можете установить её самостоятельно. Откройте окно командной строки, перейдите в папку проекта и установите vk-miniapps-deploy:

Командная строка

`cd <путь-к-папке-проекта> npm install @vkontakte/vk-miniapps-deploy --include=dev`

#### [](<#4.1. Настройте проект для работы с хостингом>)4.1. Настройте проект для работы с хостингом

1.  1.
    
    В каталоге проекта откройте файл `vk-hosting-config.json`.
    
2.  2.
    
    В поле `app_id` замените `0` на идентификатор вашего мини-приложения. Сохраните изменения.
    
    Файл vk-hosting-config.json
    
    `{ "static_path": "build", "app_id": 535577**, "endpoints": { "mobile": "index.html", "mvk": "index.html", "web": "index.html" } }`
    
    Подробнее о значениях в файле — в разделе [Файл vk-hosting-config.json](/ru/mini-apps/development/hosting/config-file).
    

#### [](<#4.2. Выложите файлы>)4.2. Выложите файлы

1.  1.
    
    Выполните следующую команду. Она запустит скрипт для размещения файлов на хостинге:
    
    Командная строка
    
    `npm run deploy`
    
    Сначала скрипт вызовет `npm run build` для сборки проекта. В результате в папке проекта появится подпапка `build`, а также архив `build.zip`. Он содержит файлы, которые будут выложены на хостинг.
    
    ![Сообщения о сборке](https://sun9-46.userapi.com/TFtXCv4Q4e9JjeKaiFsUFVFox4rNNMat8pEYxg/zt7FjepoWeQ.jpg "Сообщения о сборке")Сообщения о сборке
    
2.  2.
    
    Разрешите размещение файлов на хостинге.
    
    ![Запрос на размещение файлов на хостинге](https://sun9-1.userapi.com/q3TVxfKrGkVI29jchJY55bnCoV2cINvDyn7HUg/LhfC28jSn0o.jpg "Запрос на размещение файлов на хостинге")Запрос на размещение файлов на хостинге
    
3.  3.
    
    Далее скрипт последовательно запросит разрешения на автоматическое изменение URL в настройках размещения мини-приложения. Примите предложения:
    
    *   •
        
        `Would you like to update prod urls?` — обновить адрес мини-приложения.
        
    *   •
        
        `Would you like to update dev urls?` – обновить адрес мини-приложения в режиме разработки.
        
    *   •
        
        `Would you like to update test group url?` – обновить адрес мини-приложения для тестовой группы. Если у вас еще нет тестовых групп, изменений не произойдет.
        
    *   •
        
        `Enter test group name` – если на предыдущем шаге вы выбрали обновить адрес для тестовой группы, укажите ее название. Вы также можете не указывать название, в этом случае адрес тестовой группы не поменяется.
        
    
    ![Запросы на изменение URL](https://sun9-42.userapi.com/6GfnijXYQjL7KcGwHLgKrRQnDZvScsTtyqzVCA/82kmCNk238o.jpg "Запросы на изменение URL")Запросы на изменение URL
    
    > При каждом запуске команды для размещения файлов на хостинге, например после внесения изменений, необходимо подтверждать изменение URL. Это связано с тем, что при обновлении файлов хеш в URL приложения изменяется.
    
4.  4.
    
    При первом выполнении пакет vk-miniapps-deploy запустит процедуру получения ключа доступа. Само получение проходит автоматически. Вам необходимо только открыть URL в браузере.
    
    ![URL для получения ключа доступа](https://sun9-82.userapi.com/n2pMXpa9bDLbpaynZy_6pmsG5iLWvo0amlNLuw/u6C06aAyVpM.jpg "URL для получения ключа доступа")URL для получения ключа доступа
    
    Скопируйте URL и откройте в браузере. Вы увидите сообщение о том, что VK Mini Apps Hosting получило доступ к вашему аккаунту.
    
    Вернитесь в окно командной строки и нажмите **y** или **Enter**, чтобы подтвердить, что авторизация выполнилась успешно. Вы увидите сообщение, что ключ получен.
    
5.  5.
    
    Далее скрипт загрузит файлы на хостинг и отобразит URL, которые соответствуют файлам на хостинге.
    
    Для изменения URL, которые используются в режиме разработки, ваше подтверждение не нужно, а вот для изменения production URL — нужно. Скрипт отправит запрос на ваше мобильное устройство. Подтвердите изменения в мобильном приложении ВКонтакте.
    
    ![Результирующие URL в окне командной строки](https://sun9-69.userapi.com/ylIjW1l4R29AOSfFjXsR7un0MY7MkHqMWL7HnA/pvU3Sgxe6gI.jpg "Результирующие URL в окне командной строки")Результирующие URL в окне командной строки
    
    Так как мы разрешили автоматическое обновление URL приложения в настройках мини-приложения, то скрипт добавил эти же URL и в настройки. Убедитесь в этом сами: откройте страницу [Настройки → Размещение](/ru/mini-apps/settings/general/placement) и посмотрите URL для различных платформ.
    

#### [](<#4.3 Запустите мини-приложение>)4.3 Запустите мини-приложение

Сразу после регистрации мини-приложение доступно ВКонтакте только его создателю и администраторам. Если вы подключили мини-приложение к Одноклассникам, то там оно доступно только создателю. Запустить мини-приложение можно по прямой ссылке, но не ранее чем через 15 минут после создания ссылки на сайте Одноклассники.

Способ запуска зависит от места запуска: десктопная или мобильная версии сайтов ВКонтакте и Одноклассники или мобильное приложение для Android или iOS.

Запустить мини-приложение можно из панели управления. Проще всего это сделать в десктопной версии сайта: перейдите на страницу [Приложения](https://vk.com/away.php?to=https%3A%2F%2Fdev.vk.com%2Fru%2Fadmin%2Fapps-list) и нажмите **Настройки** напротив нужного приложения. Затем скопируйте адрес мини-приложения для нужной площадки.

![Запуск мини-приложения из панели управления](https://sun9-30.userapi.com/yUpeUwrhtXidwUO03tN0ru5ULf6hfdUjXnyzuQ/-xODWTYlfE4.jpg "Запуск мини-приложения из панели управления")Запуск мини-приложения из панели управления

Мини-приложение откроется в интерфейсе ВКонтакте или Одноклассников.

![Экран мини-приложения ВКонтакте](https://sun9-3.userapi.com/oQDWKwe8QAeVxAFQVbiQuFIS02mFHlJyDKGJyg/kCYBKRhLoBE.jpg "Экран мини-приложения ВКонтакте")Экран мини-приложения ВКонтакте

Чтобы запустить неопубликованное мини-приложение из UI мобильного приложения ВКонтакте, отправьте ссылку на мини-приложение в чат **Избранное** и запустите его по этой ссылке. Таким же способом можно запустить мини-приложение в Одноклассниках.

## [](#FAQ)FAQ

#### [](<#Не удалось выполнить инструкцию, получаю ошибки. Что делать?>)Не удалось выполнить инструкцию, получаю ошибки. Что делать?

Убедитесь, что вы используете версию [Node.js 14.18.0](/ru/mini-apps/software-installation#%D0%9A%D0%B0%D0%BA%20%D1%81%D0%BC%D0%B5%D0%BD%D0%B8%D1%82%D1%8C%20%D0%B2%D0%B5%D1%80%D1%81%D0%B8%D1%8E%20Node.js?) или более позднюю. Версия 14.18.0 — минимально необходимая версия для работы библиотеки [VK Bridge](/ru/bridge/overview). Однако [пакет create-vk-mini-app](/ru/mini-apps/getting-started/create-vk-mini-app), который используется для создания проекта мини-приложения из шаблонов, требует версию Node.js 18.0.0 или более позднюю.

Если всё равно возникают ошибки, поищите ответ в сообществе [VK Mini Apps](https://vk.com/away.php?to=https%3A%2F%2Fvk.com%2Fvkappsdev).

#### [](<#После выполнения команды создания проекта мини-приложения на Windows 10 Pro получаю ошибку. Что делать?>)После выполнения команды создания проекта мини-приложения на Windows 10 Pro получаю ошибку. Что делать?

Если вы используете операционную систему Windows 10 Pro и после выполнения команды `npx @vkontakte/create-vk-mini-app` вы получили ошибку:

1.  1.
    
    Перейдите в созданную папку проекта мини-приложения:
    
    Командная строка
    
    `cd <путь-к-папке>`
    
2.  2.
    
    Установите зависимости Node.js:
    
    Командная строка
    
    `npm install`
    
3.  3.
    
    Установите модули из файла `package-lock.json`:
    
    Командная строка
    
    `npm ci`
    

После того как вы выполните эти команды, вы сможете продолжить выполнять инструкцию.

#### [](<#Приложение не грузится до конца при локальном запуске. Почему?>)Приложение не грузится до конца при локальном запуске. Почему?

Скорее всего вы попытались открыть мини-приложение не из ВКонтакте UI по адресу, который сборщик Vite показывает в строке Local или Network.

В демонстрационных целях мини-приложение, созданное из шаблона, при старте вызывает события VK Bridge для получения информации о пользователе, который запустил мини-приложение. При запуске вне ВКонтакте этот код работать не будет. Поэтому вы видите страницу, которая не загружается до конца.

Для локальной отладки вы можете использовать библиотеку [`vk-bridge-mock`](/ru/mini-apps/development/debugging#%D0%9E%D1%82%D0%BB%D0%B0%D0%B4%D0%BA%D0%B0%20%D0%BF%D1%80%D0%B8%20%D0%BF%D0%BE%D0%BC%D0%BE%D1%89%D0%B8%20vk-bridge-mock), которая позволит эмулировать получение тестовых данных.

#### [](<#Получаю ошибку sh: 1: vk-tunnel: Permission denied. Что делать?>)Получаю ошибку `sh: 1: vk-tunnel: Permission denied`. Что делать?

Если вы выполнили команду `npm run tunnel` на Linux и видите ошибку доступа, выдайте права на выполнение исполняемому файлу библиотеки:

Командная строка

`chmod +x <полный-путь-к-папке-проекта>/node_modules/@vkontakte/vk-tunnel/bin/vk-tunnel`

#### [](<#Получаю ошибку Error: 5: User authorization failed: invalid session. Что делать?>)Получаю ошибку `Error: 5: User authorization failed: invalid session`. Что делать?

Если вы выполнили команду `npm run deploy` и видите ошибку авторизации, обновите токен библиотеки.

Windows

`del C:\Users\<имя-пользователя>\.config\configstore\@vkontakte\vk-miniapps-deploy.json`

macOS

`rm ~/.config/configstore/@vkontakte/vk-miniapps-deploy.json`

#### [](<#Получаю ошибку Error: Пользователь не является создателем приложения. Что делать?>)Получаю ошибку `Error: Пользователь не является создателем приложения`. Что делать?

Если вы выполнили команду `npm run tunnel` и видите ошибку `ERROR: Пользователь не является создателем приложения`, обновите токен утилиты:

Windows

`del C:\Users\<имя-пользователя>\.config\configstore\@vkontakte\vk-tunnel.json`

macOS

`rm ~/.config/configstore/@vkontakte/vk-tunnel.json`

> **Обратите внимание!** Если ошибка сохраняется, убедитесь, что вас добавили в [администраторы](/ru/mini-apps/settings/managers) мини-приложения.

#### [](<#Получаю ошибку Error: connect ECONNREFUSED 127.0.0.1:10888. Что делать?>)Получаю ошибку `Error: connect ECONNREFUSED 127.0.0.1:10888`. Что делать?

Убедитесь, что вы запустили проект:

Командная строка

`npm run start -- --host`

#### [](<#Какие библиотеки используются для создания мини-приложений?>)Какие библиотеки используются для создания мини-приложений?

Обязательна к использованию только одна библиотека — [VK Bridge](/ru/bridge/getting-started). Она необходима для взаимодействия с площадкой ВКонтакте и Одноклассники.

Ещё вы можете использовать следующие необязательные библиотеки:

*   •

#### [](<#Включена ли поддержка VK Bridge в VKUI?>)Включена ли поддержка VK Bridge в VKUI?

Приложение, созданное с помощью пакета create-vk-mini-app, использует [VKUI](/ru/libraries/vkui) — библиотеку React-компонентов для создания UI мини-приложений.

Ранние версии VKUI включали поддержку библиотеку [VK Bridge](/ru/bridge/overview), которая необходима для работы мини-приложений. Она устанавливалась на компьютеры разработчиков при установке пакета VKUI.

По мере своего развития VKUI превратилась в полноценную React-библиотеку, которую можно использовать не только для создания мини-приложений ВКонтакте, но и для разработки других приложений на React. Поэтому при подготовке VKUI версии 6.0 разработчики приняли решение исключить из неё поддержку VK Bridge.

Шаблоны в create-vk-mini-app настроены на использование VKUI версии 4.28. Эта версия включает поддержку VK Bridge. Поэтому мини-приложения, созданные с помощью create-vk-mini-app, работают корректно.

Если вы будете подключать VKUI самостоятельно, то скорее всего вы будете использовать последнюю версию этой библиотеки — 6.0 или более позднюю, в которой поддержка VK Bridge отсутствует. В этом случае вам надо самостоятельно подключить VK Bridge к своему проекту. Это несложно, подробные инструкции — в [документации VKUI](https://vk.com/away.php?to=https%3A%2F%2Fvkui.io%2Fintegrations%2Fvk-mini-apps).

#### [](<#Как поделиться мини-приложением с друзьями?>)Как поделиться мини-приложением с друзьями?

По умолчанию мини-приложение отключено для пользователей и видно только его создателю по ссылке `vk.com/app{ID}` ВКонтакте и `ok.ru/app/{ID}`.

Чтобы открыть мини-приложение конкретному пользователю ВКонтакте, добавьте пользователя в [администраторы мини-приложения](/ru/mini-apps/settings/managers#%D0%94%D0%BE%D0%B1%D0%B0%D0%B2%D0%BB%D0%B5%D0%BD%D0%B8%D0%B5%20%D0%B0%D0%B4%D0%BC%D0%B8%D0%BD%D0%B8%D1%81%D1%82%D1%80%D0%B0%D1%82%D0%BE%D1%80%D0%B0). Вы можете сделать его администратором с частичным доступом с правом «Тестирование». Это даст пользователю возможность работать с отключённым приложением.

Ещё возможный вариант: создать в настройках, в разделе [Тестирование](/ru/mini-apps/settings/test-groups) группу тестировщиков и включить пользователя в эту группу.

Чтобы открыть мини-приложение всем, зайдите в [Настройки → Размещение](/ru/mini-apps/settings/general/placement#%D0%A1%D0%BE%D1%81%D1%82%D0%BE%D1%8F%D0%BD%D0%B8%D0%B5%20%D0%B4%D0%BB%D1%8F%20%D0%BF%D0%BE%D0%BB%D1%8C%D0%B7%D0%BE%D0%B2%D0%B0%D1%82%D0%B5%D0%BB%D0%B5%D0%B9). Измените **Состояние для пользователей** мини-приложения на **Включено**.

В Одноклассниках приложение будет доступно другим пользователям только после прохождения [модерации](/ru/mini-apps/settings/moderation).

#### [](<#Как запустить мини-приложение в мобильной версии сайта>)Как запустить мини-приложение в мобильной версии сайта

Мини-приложение можно запустить в мобильной версии сайта по прямой ссылке.

1.  1.
    
    Укажите URL для запуска мини-приложения в настройке [Настройки → Размещение → Мобильная версия сайта → URL](/ru/mini-apps/settings/general/placement#URL).
    
2.  2.
    
    Откройте в мобильном браузере ссылку на мини-приложение:
    
    *   •
        
        `https://m.vk.com/app{ID}` для ВКонтакте.
        
    *   •
        
        `https://m.ok.ru/app/{ID}` для Одноклассников.
        
    
    где `{ID}` — идентификатор мини-приложения, который отображается в [настройках](/ru/mini-apps/settings/overview#%D0%93%D0%B4%D0%B5%20%D0%BD%D0%B0%D0%B9%D1%82%D0%B8%20ID%20%D0%BC%D0%B8%D0%BD%D0%B8-%D0%BF%D1%80%D0%B8%D0%BB%D0%BE%D0%B6%D0%B5%D0%BD%D0%B8%D1%8F).
    

#### [](<#Как запустить мини-приложение в мобильном приложении ВКонтакте>)Как запустить мини-приложение в мобильном приложении ВКонтакте

1.  1.
    
    Укажите URL для запуска мини-приложения в настройке [Настройки → Размещение → Мобильное приложение → URL](/ru/mini-apps/settings/general/placement).
    
2.  2.
    
    Откройте мобильное приложение и перейдите в сообщения.
    
3.  3.
    
    Отправьте в чат **Избранное** сообщение со ссылкой на мини-приложение:
    
    ``* `https://vk.com/app{ID}` для ВКонтакте. * `https://ok.ru/app/{ID}` для Одноклассников. где `{ID}` — [идентификатор мини-приложения](mini-apps/settings/overview#ID) из настроек.  ``
    
4.  4.
    
    Нажмите на ссылку в отправленном сообщении. Мобильное приложение запустит мини-приложение в элементе управления WebView.
    

#### [](<#Как сделать работу мини-приложения корректной на обоих доменах ВКонтакте — vk.com и vk.ru?>)Как сделать работу мини-приложения корректной на обоих доменах ВКонтакте — vk.com и vk.ru?

Некоторые разработчики проверяют входящие запросы и указывают в конфигурации мини-приложения адреса надёжных сайтов. Не забудьте добавить домен vk.ru в список доверенных сайтов. Это нужно, чтобы ваше мини-приложение работало корректно у пользователей, которые запустили его на домене vk.ru.

Если в своих мини-приложениях при отправке ответов с сервера вы используете заголовок [X-Frame-Options](https://vk.com/away.php?to=https%3A%2F%2Fdeveloper.mozilla.org%2Fen-US%2Fdocs%2FWeb%2FHTTP%2FHeaders%2FX-Frame-Options), то рекомендуем внести правки, чтобы поддержать значения `vk.ru` и `m.vk.ru`.

#### [](<#У меня возникли проблемы при разработке мини-приложения в режиме инкогнито>)У меня возникли проблемы при разработке мини-приложения в режиме инкогнито

Из-за настроек безопасности некоторые браузеры блокируют файлы cookie и объект `localStorage` в режиме инкогнито. Это может приводить к ошибкам в работе мини-приложения. Чтобы обойти ограничения, используйте специальные события библиотеки VK Bridge для хранения данных:

*   •
    
    [`VKWebAppStorageSet`](/ru/bridge/VKWebAppStorageSet)
    
*   •
    
    [`VKWebAppStorageGet`](/ru/bridge/VKWebAppStorageGet)
    
*   •
    
    [`VKWebAppStorageGetKeys`](/ru/bridge/VKWebAppStorageGetKeys)
