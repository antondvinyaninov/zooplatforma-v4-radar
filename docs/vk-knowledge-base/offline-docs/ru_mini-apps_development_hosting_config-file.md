# Файл `vk-hosting-config.json`

Конфигурационный файл `vk-hosting-config.json` хранит настройки для загрузки компонентов мини-приложения на хостинг. Эти настройки использует библиотека [vk-miniapps-deploy](https://vk.com/away.php?to=https%3A%2F%2Fgithub.com%2FVKCOM%2Fvk-miniapps-deploy).

Подробнее о размещении файлов мини-приложения на хостинге, ограничениях и возможных ошибках — в разделе [Хостинг статики для мини-приложений](/ru/mini-apps/development/hosting/overview).

## [](#Пример)Пример

JSON

`{ "static_path": "build", "app_id": 123456, "endpoints": { "mobile": "index.html", "web": "index.html", "mvk": "index.html" } }`

## [](<#Параметры конфигурационного файла>)Параметры конфигурационного файла

Поле

Описание

`static_path`

Папка, которая содержит файлы для размещения на хостинге. В примере выше это папка `build`. Указывайте путь к папке относительно файла `vk-hosting-config.json`.

`app_id`

Идентификатор мини-приложения. Идентификатор отображается в [панели управления мини-приложением](/ru/mini-apps/settings/overview#%D0%93%D0%B4%D0%B5%20%D0%BD%D0%B0%D0%B9%D1%82%D0%B8%20ID%20%D0%BC%D0%B8%D0%BD%D0%B8-%D0%BF%D1%80%D0%B8%D0%BB%D0%BE%D0%B6%D0%B5%D0%BD%D0%B8%D1%8F?). Вы можете не указывать это значение в файле `vk-hosting-config.json`. Тогда вам нужно будет сохранить идентификатор мини-приложения в переменной окружения [`MINI_APPS_APP_ID`](/ru/mini-apps/development/hosting/ci#%D0%9F%D0%B5%D1%80%D0%B5%D0%BC%D0%B5%D0%BD%D0%BD%D1%8B%D0%B5%20%D0%BE%D0%BA%D1%80%D1%83%D0%B6%D0%B5%D0%BD%D0%B8%D1%8F) до выкладки файлов на хостинг.

`endpoints`

Имена файлов загрузки вашего проекта в папке `static_path`, которые будут использоваться при запуске мини-приложения:    • `mobile` — на мобильных устройствах.    • `web` — из десктопной версии сайта.    • `mvk` — из мобильной версии сайта. При загрузке файлов на хостинг библиотека vk-mini-apps-deploy сформирует URL для запуска мини-приложения в формате: `https://<адрес_сервера>/<путь_к_файлу_загрузки>`. Путь к файлу включает его название и задаётся от корня папки `static_path`. Полученный URL также указывается в панели управления мини-приложением в разделе [**Размещение**](/ru/mini-apps/settings/general/placement).

`noprompt`

Параметр, который определяет, будет ли библиотека запрашивать подтверждения при выкладке. Если указано `1`, библиотека не задаёт вопросы в командной строке и использует значения по умолчанию. Используйте этот режим для автоматической загрузки файлов на хостинг, например из системы CI/CD. Подробнее — в разделе [Добавление файлов на хостинг из автоматических сборок](/ru/mini-apps/development/hosting/ci).

`proxy_url`

Параметр, который добавляет возможность выполнять все API-запросы через заданный прокси-сервер.
