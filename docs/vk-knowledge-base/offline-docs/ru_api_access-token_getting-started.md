# Ключи доступа

## [](<#Общие сведения>)Общие сведения

Ключи доступа используются при отправке запросов API ВКонтакте. Они помогают идентифицировать отправителя запроса и его права доступа.

Ключ необходимо указывать при вызове каждого API-запроса. Подробности — в разделе [Формат запросов](/ru/api/api-requests).

Для работы с API ВКонтакте используются сервисный ключ доступа, ключи пользователя и сообщества.

## [](<#Сервисный ключ доступа>)Сервисный ключ доступа

Этот ключ доступа идентифицирует сайт или приложение, которые отправляют запрос. Все запросы к API ВКонтакте с таким ключом считаются совершёнными от имени этого приложения.

Сервисный ключ доступа ещё называют **ключом доступа приложения**. ВКонтакте генерирует его при регистрации приложения на платформе. Разработчики могут найти его в настройке [Сервисный ключ](/ru/mini-apps/settings/development/keys#%D0%A1%D0%B5%D1%80%D0%B2%D0%B8%D1%81%D0%BD%D1%8B%D0%B9%20%D0%BA%D0%BB%D1%8E%D1%87) в панели управления приложением.

![Ключ доступа в настройках мини-приложения или игры](https://sun9-20.userapi.com/U3ZNRorrl-J_k0GEzsjl-DpqYFjCfJ-Pfc-LRg/0_gqqOxyHCA.jpg "Ключ доступа в настройках мини-приложения или игры")Ключ доступа в настройках мини-приложения или игры

Срок действия такого ключа не ограничен. Если сервисный ключ скомпрометирован, его можно обновить в настройках.

Сервисные ключи используются для игр и мини-приложений, которые регистрируются на платформе ВКонтакте.

**Важно!** В целях безопасности используйте сервисный ключ для выполнения запросов с серверной стороны вашего приложения, то есть для выполнения back-to-back запросов. Не передавайте и не храните этот ключ в клиентской части.

## [](<#Ключ доступа пользователя>)Ключ доступа пользователя

Этот ключ доступа сообщает серверам ВКонтакте, от имени какого пользователя выполняется запрос и какими [правами доступа](/ru/reference/access-rights) он обладает.

Способ получения таких ключей зависит от типа приложения:

*   •
    
    [Мини-приложения и игры ВКонтакте](/ru/general/app-types#%D0%92%D1%81%D1%82%D1%80%D0%B0%D0%B8%D0%B2%D0%B0%D0%B5%D0%BC%D1%8B%D0%B5%20%D0%BF%D1%80%D0%B8%D0%BB%D0%BE%D0%B6%D0%B5%D0%BD%D0%B8%D1%8F).
    
    Получить ключ доступа пользователя можно с помощью события [`VKWebAppGetAuthToken`](/ru/bridge/VKWebAppGetAuthToken) библиотеки [VK Bridge](/ru/bridge/overview).
    
*   •
    
    [Standalone-приложения и сайты](/ru/general/app-types#Standalone-%D0%BF%D1%80%D0%B8%D0%BB%D0%BE%D0%B6%D0%B5%D0%BD%D0%B8%D0%B5).
    
    Получить ключ доступа пользователя можно программно с помощью библиотеки VK ID SDK, либо через специальный URL. Подробнее — в документации VK ID:
    
        ► [Получение ключа доступа пользователя](https://vk.com/away.php?to=https%3A%2F%2Fid.vk.com%2Fabout%2Fbusiness%2Fgo%2Fdocs%2Fru%2Fvkid%2Flatest%2Fvk-id%2Fconnection%2Ftokens%2Faccess-token%23Kak-poluchit-token)
    
    > До 25 июня 2024 года ВКонтакте предлагал два способа получения ключа доступа пользователя: Implicit Flow и Authentication Code Flow. Сейчас эти способы устарели. Для получения ключей следуйте инструкции по ссылке выше.
    
    Обратите внимание, что при запросе ключа вы должны указать типы данных пользователя в настройках приложения, к которым оно будет получать доступ:
    
    *   •
        
        **Базовые** — фамилия и имя, фото профиля, пол и дата рождения, а также почта по запросу.
        
    *   •
        
        **Расширенные** — номер телефона.
        
    
    Работать с базовыми доступами можно сразу после создания приложения в [сервисе авторизации VK ID](https://vk.com/away.php?to=https%3A%2F%2Fid.vk.com%2Fabout%2Fbusiness%2Fgo%2F). Чтобы работать с расширенными доступами, подтвердите профиль бизнеса. Подробнее о работе с доступами — [в документации VK ID](https://vk.com/away.php?to=https%3A%2F%2Fid.vk.com%2Fabout%2Fbusiness%2Fgo%2Fdocs%2Fru%2Fvkid%2Flatest%2Fvk-id%2Fconnection%2Fcreate-application%23Rasshirennye-dostupy).
    
    ![Доступы в настройках приложения](https://sun9-61.userapi.com/fFDlRmwKJ6gNyBQPo55u4tLpLqbYVAtyMTvg1A/VIaYSS7ju3s.jpg "Доступы в настройках приложения")Доступы в настройках приложения
    
    Не получится сгенерировать ключ, который даёт доступ, например, к телефону пользователя, если такой вид доступа не выбран в настройках приложения.
    
    Помимо указанных доступов есть [расширенные](/ru/reference/access-rights): к списку друзей, видеозаписям, фотографиям и другим данным. Получение расширенных доступов рассматривается в индивидуальном порядке. Чтобы их получить, напишите на почту [devsupport@corp.vk.com](mailto:devsupport@corp.vk.com).
